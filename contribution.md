# Contributing to WAMS Woo Notifier

Thank you for your interest in contributing to WAMS Woo Notifier! We welcome contributions from the community to help make this plugin better for everyone.

## Ways to Contribute

There are many ways you can contribute to the project:
*   **Reporting Bugs:** If you find a bug, please open an issue and provide as much detail as possible (WordPress version, WooCommerce version, PHP version, and steps to reproduce).
*   **Requesting Features:** Have an idea for a new feature? Let us know by opening a feature request.
*   **Submitting Pull Requests:** If you want to fix a bug or implement a feature yourself, we'd love to see your code!
*   **Translations:** Help us translate the plugin into more languages.

## Getting Started

1.  **Fork the Repository:** Create your own fork of the repository on GitHub.
2.  **Clone the Fork:** `git clone https://github.com/your-username/wams-woo-notifier.git`
3.  **Install Dependencies:** Run `composer install` to install any necessary development dependencies.
4.  **Make Your Changes:** Create a new branch for your changes (`git checkout -b feature/my-new-feature`).

## Coding Standards

To maintain code quality and consistency, we follow the [WordPress Coding Standards](https://developer.wordpress.org/coding-standards/). Please ensure your code adheres to these guidelines.

*   Use tabs for indentation.
*   Use `camelCase` for variable and function names (or `snake_case` if following WordPress hooks/filters).
*   Document your code using PHPDoc comments.
*   Ensure your code is secure and follows WordPress security best practices (escaping, sanitization, and nonces).

## Pull Request Process

1.  **Update Documentation:** If your changes affect how the plugin is used, please update the `README.md` or `usecase.md` accordingly.
2.  **Tested and Verified:** Ensure your changes are thoroughly tested and do not introduce any regressions.
3.  **Clear PR Description:** Provide a clear and concise description of the changes you've made and why they are necessary.
4.  **Review:** Once you submit your PR, it will be reviewed by the maintainers. Be prepared to address any feedback or suggestions.
